# Tane Money ローンチ前チェックリスト

コード側の対策はデプロイ済みでも、**以下のFirebaseコンソール作業を完了するまで一般公開しないこと**。
（現状はFirestoreがルール無しで全開放の可能性が高く、他人の家族データが読み書きできてしまう）

## 必須：Firebaseコンソール作業（所要 約10分）

https://console.firebase.google.com/project/tane-money

### 1. 匿名認証を有効化（先にやる。順番厳守）
- Authentication → Sign-in method → 「匿名」を**有効**にする
- ※ルールより先に有効化しないと、ルール適用後に全ユーザーがアクセス不能になる

### 2. Firestoreセキュリティルールをデプロイ
- Firestore Database → ルール → リポジトリの `firestore.rules` の内容を貼り付けて「公開」
- またはローカルで: `firebase deploy --only firestore:rules`
- 適用後、アプリで読み書きできることを必ず実機確認（匿名認証が効いていれば動く）

### 3. App Check を有効化（推奨・総当たり対策の仕上げ）
- App Check → アプリを登録（reCAPTCHA v3 / Enterprise）
- まず「モニタリングモード」で開始し、正常を確認してから「適用」に切り替え

### 4. 承認済みドメインの確認
- Authentication → Settings → 承認済みドメイン を
  `tane-money.vercel.app`（+開発用localhost）のみに絞る

## コード側で対策済み（v2026-06-12）

| 項目 | 内容 |
|---|---|
| 認証 | 匿名認証を自動サインイン（Auth未有効でも従来動作にフォールバック） |
| ファミリーコード | 暗号乱数 TANE-XXXX-XXXX（31^8≈8500億通り）。旧4文字コードも引き続き使用可 |
| PIN | SHA-256ハッシュで保存（平文PINはクラウドに残らない。旧データは初回ロードで自動移行） |
| コード変更 | 形式検証（8文字以上）＋確認ダイアログ。短い語呂コードの新規作成は不可 |
| ログ肥大 | 4000件超でドキュメント側は「くりこし」集約（1MB上限対策。全ログはサブコレクションに保存） |
| 新規導線 | 旧FamilySetupゲート廃止 → SetupWizardが正しく表示。参加/作成後に同期が起動 |
| PWA | PNGアイコン(192/512/maskable/apple-touch)、SW v5でプリキャッシュ |

## 既知の制限（ローンチ後の課題）

- ファミリーコードを知っている人は誰でも参加できる（招待制ではない）。
  本格的なメンバーバインド（UID登録制）は次フェーズ
- 4桁PINはハッシュ化しても総当たり耐性はない（端末ロック用途。データ保護はルールが担う）
- AI分析機能はクライアント直叩きでAPIキーが無いため本番では動かない（→無効化済みか要確認）
- 株価・為替は非公式プロキシ経由。取得失敗時は「サンプル値」表示に切替済み
